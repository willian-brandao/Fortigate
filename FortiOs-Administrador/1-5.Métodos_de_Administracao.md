# Métodos de Administração 

<img width="1007" height="544" alt="image" src="https://github.com/user-attachments/assets/7b5c1dfd-9aa0-4040-9a1e-1b6dcbbbb8cd" />

Várias funcionalidades estão disponíveis na GUI e CLI do Fortigate, mas existem algumas exceções. Não é possível visualizar reports na CLI. Também, as configurações mais avançadas e comandos de diagnósticos para super usuários usualmente só estão disponíveis no GUI. 

À medida que você se familiariza com o FortiGate — especialmente se quiser criar scripts para sua configuração — talvez seja interessante usar a CLI, a API ou o FortiManager, além da GUI. Você pode acessar a CLI pelo widget JavaScript na GUI chamado CLI Console, ou por meio de um emulador de terminal como Tera Term ou PuTTY. Seu emulador de terminal pode se conectar pela rede — via SSH — ou pela porta de console local.

O SNMP e alguns outros protocolos administrativos também são suportados, mas são somente leitura. Você não pode usá-los para a configuração básica.


## Perfis de Administradores

<img width="1029" height="575" alt="image" src="https://github.com/user-attachments/assets/69725b1d-61a6-4cc1-928b-b297d15e8a3c" />

Quando atribuímos permissões para o perfil de administrador, pode-se especificar leitura-e-escrita, apenas-leitura, ou nada em nenhuma área. 

Por padrão, existem perfis especiais nomeados super_admin, que é usado pela conta nomeada admin. Não se pode mudar isso. Isso provê acesso completo para tudo, fazendo a conta admin similar a conta de root de superusuário. 

Você não pode requisitar para usar o perfil padrão. Pode-se criar um perfil criado como acesso_auditor com permissão de apenas-leitura. Restringindo as permissões de pessoas para as necessidades deles ou do trabalho deles é uma boa prática, porque se a conta for comprometida, o comprometimento para o dispositivo do FortiGate não irá afetar tudo. Para fazer isso, criar perfis de administradores e em seguida, selecionar os perfis apropriados quando configurar uma conta. 

A opção "Override Idle Timeout" permite o valor admintimeout, abaixo de "config system accprofile", para ser sobrescrito por cada perfil de acesso. Pode-se configurar o perfil de administrador para aumentar o tempo de inatividade e facilitar o uso de GUI para monitoramento centralizado. 

Os efeitos dos perfis de administradores são:

* Controle granular do tempo de inatividade: Cada perfil de administrador pode ter seu próprio **admin-timeout**, permitindo que diferentes usuários ou funções permaneçam logados por períodos distintos.
* Melhor usabilidade para monitoramento: É possível configurar tempos maiores para perfis usados em monitoramento central (GUI/CLI), evitando logouts frequentes.
* Maior segurança: Temos mais restritivos podem ser aplicados a perfis sensíveis, reduzindo o risco de sessões abandonadas.
* Evita impacto global: As alterações se aplicam apenas ao perfil selecionado, evitando mudanças não intencionais para todos os administradores.
* Gerenciamento baseado em funções: Permite alinhar o comportamento do timeout com a função exercida (ex: monitoramento NOC vs acesso administrativo eventual).

Atualmente, dependendo do tipo de perfil de administrador que é atribuído, um administrador pode permitir o acesso para todo o dispositivo Fortigate. Por exemplo, pode-se confiigurar uma conta que pode-se visualizar apenas configurações de vpns e configurar outro perfil para criar políticas de firewall e perfis de segurança.
