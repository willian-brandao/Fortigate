# Modos de Operação 

Quando o FortiGate é implementado, pode-se escolher entre dois modos de operação: Modo NAT ou Modo Transparente.

No modo NAT, FortiGate roteia pacotes baseados na camada 3, como um roteador. Cada uma dessas interfaces lógicas tem um endereço IP e o FortiGate determina as interfaces de entrada e saída baseado no endereço IP de destino e nas entradas da tabela de roteamento. 

No modo transparente, FortiGate direciona pacotes de camada 2, como um switch. Essa interface não tem um endereço IP e o FortiGate identifica as interfaces de entrada e saída baseado no endereço MAC de destino. O dispositivo em modo transparente tem um endereço IP usado para gerenciar o tráfego.

O Modo NAT é modo de operação padrão.

Quando o FortiGate está operando em modo NAT, cada interface que contém tráfego deve ter um endereço IP. Quando está em modo NAT, FortiGate pode usar o endereço IP de origem do tráfego, se isso precisa começar ou responder a uma sessão, e como o endereço de destino para os dispositivos tentam conectar o FortiGate ou rotear tráfego através dele. Existem múltiplos caminhos para conseguir um endereço IP: 
* manualmente
* automaticamente, usando o DHCP ou protocolo ponto a ponto sobre Ethernet (PPPoE).

Configurações de Funções de Interfaces tipicamente definidas e agrupadas juntas:
* Prevenir perda de configuração acidental.
  * Quatro Tipos:
    * LAN
    * WAN
    * DMZ
    * Indefinido
  * Nenhum na lista de políticas.
* Alias é uma descrição amigável para uma interface:
  * Usado em lista de políticas para identificar interfaces para um propósito.

Quanta vezes os times vêem problemas de redes causados por um servidor DHCP habilitado na interface de WAN?

Pode-se configurar uma função para uma interface. As função mostradas na GUI são as configurações usuais de interfaces para cada parte da topologia. Configurações que não são aplicadas para a regra atual ficam escondidas na GUI. Isso previne perda de configuração acidental. 

Por exemplo, quando a regra confgurada como WAN, não existe servidor DHCP e configuração de detecção de dispositivo definidas. Detecção de dispositivos é usado para detectar dispositivos internamente na LAN. 

Se existe um caso não usual e necessita-se usar uma opção escondida pela regra atual, pode-se sempre mudar a regra para não definida. Isso mostra todas as opções. Para ajudar a lembrar o propósito de cada interface, pode ser atribuir "alias". Por exemplo, pode-se chamar a porta3 de "rede_interna". Isso faz com que a lista de políticas seja fácil de compreender. 

Opções de acesso administrativo também podem ser limitadas dependendo da regra que foi selecionada para a interface. Por exemplo, a configuração da interface para WAN pode não mostrar a opção PING mitigando o risco de resposta a um ataque ICMP DoS vindo da WAN.
